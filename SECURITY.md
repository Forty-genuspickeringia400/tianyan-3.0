# Security Policy

## Reporting a vulnerability
If you discover a security issue, please do not publish it immediately in a public issue.

Please contact the maintainer privately first and include:
- a clear description of the issue
- affected area
- reproduction steps if possible
- expected impact

## Scope
This project is a public runtime prototype. Please report issues related to:
- unsafe boundary handling
- verification bypasses
- privilege escalation paths
- unintended remote execution exposure
- sensitive data leakage

## Notes
Do not include real secrets, tokens, or private credentials in reports.
