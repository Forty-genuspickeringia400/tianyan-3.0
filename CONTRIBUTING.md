# Contributing

Thanks for your interest in Tianyan 3.0.

## Scope
This public repository is intended to expose the runnable runtime prototype clearly and safely.
Please keep contributions aligned with:
- runtime architecture clarity
- observability and verification discipline
- safety boundaries
- governed evolution

## Basic flow
1. Fork the repository
2. Create a feature branch
3. Keep changes scoped and explain why they matter
4. Run available tests before opening a PR
5. Open a pull request with a clear summary

## Contribution priorities
- docs clarity
- examples and quickstart improvements
- bug fixes in the public runtime path
- tests and smoke validation
- dashboard/workbench readability

## Things to avoid
- adding secrets or machine-specific configuration
- overstating unverified capabilities
- weakening safety boundaries without strong justification
- mixing unrelated refactors into one PR
